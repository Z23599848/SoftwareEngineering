VS Code and HTML Quick start

  1. Open VS Code
  2. Access the start page as blow. You may need to choose ‘file’ -> ‘new window’ in the top menu

  1. Click ‘open folder’
  2. In the file explorer, create a new folder called SEModule in a location of your choice
  3. Click ‘open’
  4. You will ask if you ‘trust the authors’ of the file. Click ‘yes’
  5. In the left hand column - the ‘explorer’ - you will see your folder. The explorer is where you can manage the files for your VS Code project
  6. Roll your mouse over the folder name until you see the new file icon appear as below

  1. Click on the new file icon and create a file called ‘test.html’
  2. The file will open in the main window
  3. Add the following HTML content to the file

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
</head>
<body>
    <h1>Welcome to My Webpage</h1>
</body>
</html>

Save the file - VS code generally does NOT have autosave

  1. Now view the HTML code rendered in the browser. To do this,
    • open a browser window.
    • Type ctrl o (both keys held down at the same time)
    • Navigate to the .html file you have created
    • You should see the page rendered in your browser window
  2. Test your setup by changing the text from ‘welcome to my webpage’ to ‘welcome to my software engineering page’ or any other text of your choice.
  3. Now go back to the browser window. DO NOT OPEN A NEW WINDOW - you can simply refresh to see your changes.

Developer Tools

Using the browser developer tools (found in all browsers, but chrome is most commonly used) to debug and refine your front-end code is an essential skill.

Try it now:

In your browser window, right click in the page and choose ‘inspect’.
A set of windows will appear, probably at the bottom or perhaps to the right of your screen. Here you will be able to examine the source code of your own page and ensure that it is as you expect. You can even find you can edit the markup, content and styles.

Note that you can do this for any web page!!. Your changes will only be visible on your own browser and will disappear after you refresh the page.

Reference: https://developer.chrome.com/docs/devtools/overview/

Headings and Paragraphs

Expand the previous example with multiple headings and paragraphs. Notice that various heading levels to emphasize the importance of hierarchy.


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Heading Levels</title>
</head>
<body>
    <h1>Main Heading (Level 1)</h1>
    <p>This is the main content of the page.</p>
    <h2>Subheading (Level 2)</h2>
    <p>Additional text for the subheading.</p>
    <h3>Sub-subheading (Level 3)</h3>
    <p>More text for the sub-subheading.</p>
</body>
</html>

Hyperlinks

Create hyperlinks to navigate between pages. NOTE that you will first need to create page1.html and page2.html. These will need to be in the same directory as your index.html file.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Hyperlinks</title>
</head>
<body>
    <h1>Visit Other Pages</h1>
    <ul>
        <li><a href="page1.html">Page 1</a></li>
        <li><a href="page2.html">Page 2</a></li>
    </ul>
</body>
</html>

To master both images and hyperlinks, you will need to develop a good understanding of absolute and relative paths:

see [W3 schools resource on file paths in HTML] (https://www.w3schools.com/html/html_filepaths.asp)

Most errors in linking to other pages or images are caused by errors in the file path in the src or href attribute.

Tables

Create a table with some data. Note how the tags here need to be ‘nested’ correctly

    <meta charset="UTF-8">
    <title>Table Example</title>
